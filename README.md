# Interactive Lightsaber Training Game

An interactive two-node system where a player uses a physical lightsaber controller to block incoming projectiles generated by a training droid.

## Overview
This project combines Arduino and Processing to create a physical game inspired by lightsaber training.

It consists of two communicating nodes:

- **Lightsaber Node** (this system): handheld controller that detects orientation and player input  
- **Training Droid Node**: generates projectile directions and controls game flow  

The droid sends a direction (1–5), and the player must match it by rotating the lightsaber. The game ends after three failed blocks.

## System Components

### Lightsaber Node
- Accelerometer (orientation detection via I2C)  
- Photo interrupter (activation button)  
- RGB LED strip (visual feedback)  
- Audio feedback via Processing  
- Serial communication (Arduino ↔ Processing)  

### Training Droid Node
- Servo motor (arm indicating direction)  
- LEDs (visual direction cues)  
- Ultrasonic sensor (start game based on distance)  
- Accelerometer (difficulty selection via lever)  
- TCP communication with lightsaber node  

## Features
- Motion-based gameplay  
- Physical interaction (controller + droid)  
- Real-time communication between nodes  
- Adjustable difficulty  
- Combined visual and audio feedback  

## Tech Stack
- Arduino (C/C++)  
- Processing 
- I2C (accelerometers)  
- Serial communication  
- TCP networking  

## Requirements
- 2× Arduino-compatible boards  
- 2× Accelerometers (e.g. ADXL345)  
- Photo interrupter sensor  
- Ultrasonic distance sensor  
- Servo motor  
- RGB LED strip + LEDs  
- Computer running Processing  
- USB connections  
- Network connection between nodes
